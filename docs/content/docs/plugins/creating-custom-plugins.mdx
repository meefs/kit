<Callout type="warn" title="Planned Content">
**Status:** New page

This page will cover how to write custom plugins: the `ClientPlugin<TInput, TOutput>` type signature, using `extendClient(client, additions)` to add capabilities (preserves property descriptors, freezes result), simple synchronous plugins (`(client) => extendClient(client, { myFeature: ... })`), async plugins (`async (client) => extendClient(client, { data: await fetch(...) })`), enforcing prerequisites via TypeScript generics (`<T extends ClientWithRpc>(client: T) => ...`), implementing standard interfaces (`ClientWithPayer`, `ClientWithTransactionSending`, etc.) for interoperability, a full worked example building a custom plugin from scratch, and best practices for naming, documentation, and publishing to npm.

It will also include a section on creating **plugin bundles**: reusable plugins that compose several smaller plugins into one setup step. This section will show how official bundles such as `solanaRpc`, `solanaDevnetRpc`, `solanaLocalRpc`, and `litesvm` are structured, when an application or library should create its own bundle, and how to keep bundles composable by accepting an existing client from `createClient()` rather than hiding client construction behind a factory.

**Draws from:** `@solana/plugin-core` source and docs (`createClient`, `extendClient`, `ClientPlugin`), kit-plugins main README "Create Your Own Plugins" and "Compose Granular Plugins" sections.

**Links to:** [Plugins — Overview](/docs/plugins) for the client model, [Available Plugins](/docs/plugins/available-plugins) for existing plugin bundles and granular plugins.
