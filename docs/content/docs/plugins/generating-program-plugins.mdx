This page will cover what Codama is (an IDL-based code generation tool for Solana programs), how Codama generates Kit-compatible program clients with plugin support, the generated plugin pattern (`tokenProgram()` → adds `client.token.accounts` and `client.token.instructions` namespaces), a brief example of generating a plugin for a custom program from an IDL, configuration options for the Codama JS renderer, publishing your generated program plugin for others to use, and links to full Codama documentation. This page is intentionally brief — Codama has its own comprehensive docs. The focus here is on how generated plugins fit into the Kit plugin ecosystem.

**Draws from:** [Codama](https://github.com/codama-idl/codama) docs, [Codama JS renderers](https://github.com/codama-idl/renderers-js) docs.

**Links to:** [Using Program Plugins](/docs/guides/using-program-plugins) for consuming generated plugins, [Available Plugins](/docs/plugins/available-plugins) for the directory of existing program plugins.
