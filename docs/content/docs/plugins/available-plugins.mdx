<Callout type="warn" title="Planned Content">
**Status:** New page

This page will have one unified table listing all known plugins (Kit-maintained and community, mixed together) with columns for name, package, a brief description, and a link to documentation. Plugins may be loosely tagged by type (e.g. "Bundle", "RPC", "Signer", "Program", "Testing", "General") but these are soft labels, not rigid categories — some plugins span multiple concerns.

The official plugin sections will cover the higher-level bundle plugins — `solanaRpc`, `solanaMainnetRpc`, `solanaDevnetRpc`, `solanaLocalRpc`, and `litesvm` — alongside the granular plugins they compose, such as `solanaRpcConnection`, `rpcAirdrop`, `rpcTransactionPlanner`, `rpcTransactionPlanExecutor`, `signer`, `payer`, `identity`, `generatedSigner`, `generatedPayer`, `airdropSigner`, `litesvmConnection`, and `planAndSendTransactions`.

Below the table, a dedicated **Program plugins** section will explain the Codama-generated pattern since program plugins are a distinct concept: they add typed program namespaces such as `client.system` or `client.token`, with `.accounts` and `.instructions` helpers below each namespace. This section absorbs the current [Compatible Clients](/docs/compatible-clients) table of `@solana-program/*` packages.

A community plugins section will encourage ecosystem contributions and explain how to get listed.

**Draws from:** Kit-plugins main README plugin table, individual plugin package READMEs, current `compatible-clients.mdx`.

**Links to:** [Creating Custom Plugins](/docs/plugins/creating-custom-plugins), [Generating Program Plugins](/docs/plugins/generating-program-plugins).

</Callout>
